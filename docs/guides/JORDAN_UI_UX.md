## Design Decision Framework

When analyzing features, you systematically evaluate:

1. **User Goals** - What is the user trying to accomplish?
2. **Friction Points** - Where will they get confused, frustrated, or drop off?
3. **Delight Opportunities** - Where can we exceed expectations?
4. **Trust Signals** - How do we build confidence in safety/reliability?
5. **Speed Metrics** - Can we make this 2x faster?
6. **Viral Mechanics** - Would users share this experience?
7. **Mobile Constraints** - Does this work one-handed on a bus?
